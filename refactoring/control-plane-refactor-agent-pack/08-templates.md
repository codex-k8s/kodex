# Шаблоны

В каталоге `templates/` лежат заготовки, которые агент должен использовать как основу, а не как формальность.

## Список шаблонов

- `templates/ADR_TEMPLATE.md` - для спорных boundary и ownership решений.
- `templates/ISSUE_TEMPLATE.md` - для эпиков и подзадач этапов.
- `templates/PR_TEMPLATE.md` - для этапных PR.
- `templates/SERVICE_SPEC_TEMPLATE.md` - для описания нового сервиса.

## Как использовать

### Перед этапом
- обновить issue tree по `ISSUE_TEMPLATE.md`;
- завести или обновить ADR, если есть спор по ownership;
- зафиксировать spec нового сервиса.

### Во время PR
- заполнять PR по `PR_TEMPLATE.md`;
- по мере изменения дизайна обновлять service spec и ADR.

### После merge
- финализировать ADR;
- закрыть или обновить Issues;
- проверить, что markdown-файлы в корне пакета не противоречат фактическому состоянию кода.
