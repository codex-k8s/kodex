# Проекты и репозитории

Статус: `approved`

## Назначение
Экран управления проектами платформы и подключёнными репозиториями, их readiness, доступами и состоянием синхронизации.

## Главные сущности
- проект
- репозиторий
- readiness
- webhook
- flow
- risk class
- участник проекта

## Основные блоки
- верхняя лента summary-card по проектам, репозиториям, подключениям и ошибкам синхронизации
- таблица проектов
- правая панель деталей выбранного проекта
- таблица репозиториев выбранного проекта
- основные действия подключения репозитория и создания проекта

## Ключевые действия
- создать проект
- подключить репозиторий
- открыть проект
- добавить репозиторий в проект
- проверить доступы
- открыть состояние readiness и webhook

## Источники данных
- платформа: project catalog, repository registry, readiness projection, memberships, flow usage
- провайдер: repository metadata, webhook state, default branch

## Что намеренно не показывается
- чат как основной режим экрана
- канбан или свободный canvas
- вымышленные сущности поверх проекта и репозитория
- diff review и длинные operational logs
