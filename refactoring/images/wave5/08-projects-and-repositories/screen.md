# Проекты и репозитории

Статус: `approved`

## Назначение
Экран управления проектами платформы, репозиториями кода, источниками продуктовой и руководящей документации, правилами подготовки `workspace` для агентов и релизной политикой проекта.

## Главные сущности
- проект
- репозиторий
- источник документации
- `workspace` composition
- release line
- release policy
- readiness
- webhook
- flow
- risk class
- участник проекта

## Основные блоки
- верхняя лента summary-card по проектам, репозиториям кода, источникам документации и проблемам
- таблица проектов
- правая панель деталей выбранного проекта
- таблица репозиториев и источников документации выбранного проекта
- вкладки репозиториев, документации, `workspace` и релизной политики
- основные действия подключения репозитория и создания проекта

## Семантика вкладок
- `Обзор` — список проектов, краткое состояние выбранного проекта и сводка подключённых источников;
- `Репозитории` — репозитории кода, `webhook`, риск и связи с сервисами;
- `Документация` — продуктовая, сервисная и руководящая документация, объявленная через `services.yaml`;
- `Рабочая область` — состав локального `workspace`, который получит агент при запуске;
- `Релизная политика` — release lines, branch rules, gates, расписание и trigger-driven automation.

## Ключевые действия
- создать проект
- подключить репозиторий
- открыть проект
- добавить репозиторий в проект
- проверить доступы
- открыть состояние readiness и webhook
- собрать `workspace`
- проверить источники документации
- создать релизную задачу
- проверить релизную политику

## Источники данных
- платформа: project catalog, repository registry, documentation source registry, workspace projection, release policy, readiness projection, memberships, flow usage
- провайдер: repository metadata, webhook state, default branch
- репозитории проекта: `services.yaml`, product docs, service docs, guiding packages

## Что намеренно не показывается
- чат как основной режим экрана
- канбан или свободный canvas
- вымышленные сущности поверх проекта и репозитория
- diff review и длинные operational logs
- значения секретов, токенов и приватных настроек доступа
