# Внешние аккаунты, интеграции и MCP

Статус: `approved`

## Назначение
Экран настройки внешних аккаунтов, provider-интеграций, лимитов и проблем авторизации с отдельной категорией MCP и явной моделью вкладок внутри выбранного provider.

## Главные сущности
- внешний аккаунт
- provider integration
- MCP-сервер
- account policy
- role binding
- rate limit
- auth issue

## Основные блоки
- верхняя лента summary-card по аккаунтам, интеграциям, проблемам авторизации и лимитам
- список категорий интеграций
- центральная вкладочная поверхность выбранного provider
- правая панель здоровья GitHub-интеграции, последних auth issues и лимитов

## Семантика вкладок
- `Аккаунты` — inventory GitHub-аккаунтов, scopes, статусы и auth state;
- `Права` — policy layer: какие роли и проекты используют какие GitHub-аккаунты и для каких операций;
- `Ограничения` — rate limits, quotas, restrictions и деградация;
- `История` — auth failures, token rotation и изменения policy.

## Ключевые действия
- добавить аккаунт
- проверить авторизацию
- открыть политику MCP
- настроить аккаунт во вкладке `Аккаунты`
- отредактировать policy layer во вкладке `Права`
- перейти к политике MCP или отдельной категории MCP-серверов
- посмотреть ограничения и квоты

## Источники данных
- платформа: provider-hub, account registry, provider policy, MCP policy, rate-limit projection, auth issues
- внешние системы: GitHub, GitLab, OpenAI / Codex, каналы уведомлений

## Что намеренно не показывается
- сохранение секретов как читаемого контента экрана
- чат и голос как основной режим
- хаотичное смешение provider settings, runtime и release-диагностики
- общий блок MCP-серверов внутри provider-экрана, если активна конкретная категория вроде GitHub
