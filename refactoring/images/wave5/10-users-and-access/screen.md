# Пользователи и доступы

Статус: `approved`

## Назначение
Экран управления пользователями платформы, их глобальными ролями, доступом к проектам и отдельным репозиториям.

## Главные сущности
- пользователь
- глобальная роль
- project membership
- repository access
- access request
- allowlist email

## Основные блоки
- верхняя лента summary-card по пользователям и типам доступа
- таблица пользователей
- правая панель профиля выбранного пользователя
- блок последних изменений доступа
- блок ожидающих подтверждений
- пояснение по GitHub OAuth и allowlist email

## Ключевые действия
- пригласить пользователя
- назначить доступ
- открыть членство в проектах
- открыть доступ к репозиториям
- перейти к ожидающим запросам
- настроить allowlist email

## Источники данных
- платформа: user catalog, memberships, access requests, audit projection
- внешние системы: GitHub OAuth identity

## Что намеренно не показывается
- самостоятельная регистрация пользователей
- HR-профили вне контекста платформы
- чат и голос как основной режим экрана
