# Каталог плагинов и пакетов документации

Статус: `approved`

## Назначение
Экран управления встроенными, внешними и пользовательскими пакетами платформы: плагинами, адаптерами, MCP-серверами и пакетами руководящей документации для агентов.

Экран должен показывать не маркетплейс ради маркетплейса, а операционный контур установки:
- из какого репозитория берётся пакет;
- какая версия проверена платформой;
- какие API платформы нужны пакету;
- какие секреты нужно заполнить до запуска;
- как пакет будет развёрнут в Kubernetes;
- где пакет применяется в проектах, flow, ролях и каналах связи.

## Главные сущности
- запись каталога пакетов
- пакет-плагин
- пакет руководящей документации
- версия пакета
- проверка версии
- manifest YAML
- Kubernetes deployment
- схема секретов
- политика прав пакета

## Основные блоки
- верхняя лента summary-card по пакетам, проверенным версиям и проблемам
- фильтры по типу, источнику, цене, проверке и совместимости
- таблица или список пакетов
- инспектор выбранного пакета
- вкладки плагинов, документации, версий, прав/секретов и установки
- блок правил безопасности каталога

## Семантика вкладок
- `Плагины` — список пакетов-плагинов, адаптеров, MCP-серверов и action-пакетов;
- `Документация` — пакеты руководящей документации, которые подключаются к проектам и подгружаются агентам в `workspace`;
- `Версии` — проверенные и непроверенные версии выбранного пакета, совместимость и откат;
- `Права и секреты` — требуемые API платформы, Human gate и форма заполнения секретов без отображения значений;
- `Установка` — параметры установки, целевой namespace, Kubernetes-ресурсы, dry-run и статус развёртывания.

## Ключевые действия
- добавить пользовательский пакет
- установить проверенную версию пакета
- проверить manifest YAML
- проверить совместимость версии
- подключить пакет документации к проекту
- заполнить секреты перед установкой
- открыть аудит установки
- откатиться к предыдущей версии

## Источники данных
- платформа: package catalog, package versions, verification records, install records, policy records, audit log
- репозитории пакетов: manifest YAML, Dockerfile, Kubernetes manifests, документация пакета
- Kubernetes: состояние deployment, service account, namespace и secret references

## Что намеренно не показывается
- значения секретов и токенов
- запуск непроверенной версии без явного Human gate
- редактирование исходного кода пакета внутри консоли
- смешение каталога пакетов с настройкой конкретного внешнего аккаунта
