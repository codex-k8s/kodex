# Исполнения и среда

Статус: `reviewing`

## Назначение
Главный операционный экран для диагностики agent `run`, platform `job`, slot и очереди ожидания без превращения UI в сырое log-view.

## Главные сущности
- `run`
- `job`
- slot
- очередь ожидания
- связанный `Issue`
- связанный `PR/MR`

## Основные блоки
- верхняя лента счётчиков по `run`, `job`, slot и очереди
- основной список/таблица исполнений с фильтрами
- правая панель деталей выбранного элемента
- карточка slot-контекста и связанных jobs
- короткий хвост лога или error excerpt

## Ключевые действия
- открыть связанный `Issue`
- открыть связанный `PR/MR`
- перейти к `job`
- возобновить или повторно запустить выполнение
- открыть связанный slot
- посмотреть причину ожидания в очереди

## Источники данных
- платформа: `runtime-manager`, `operations-hub`, queue projections, slot state, job lineage
- провайдер: ссылки на связанные `Issue` и `PR/MR`

## Что намеренно не показывается
- длинные сырые логи как основной контент экрана
- чат как основной рабочий режим
- свободный graph/canvas
