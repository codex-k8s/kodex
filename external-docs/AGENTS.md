# Инструкции для агентов в `external-docs/**`

## Назначение

`external-docs/**` содержит необязательные submodule-ссылки на внешние репозитории документации и пакетов. Это не место для копирования содержимого внешних репозиториев в `kodex`.

## Правила работы

- Обычная работа в `kodex` не должна требовать инициализации submodule.
- Приватные submodule инициализировать только при явной необходимости и наличии доступа.
- Не запускать массовый `git submodule update --init --recursive` без причины: приватные источники могут быть недоступны другим участникам.
- Если меняется содержимое внешнего репозитория, сначала commit/push делается внутри этого внешнего репозитория, затем в `kodex` обновляется только gitlink.
- Изменения в содержимом внешнего репозитория вносятся через отдельную ветку и PR этого репозитория; обновление gitlink в `kodex` делается уже после merge.
- Если меняется активная локальная копия гайдов в `docs/design-guidelines/**`, проверить, нужно ли синхронизировать соответствующий публичный репозиторий в `external-docs/guidelines/**`.
- Если меняется активная локальная копия шаблонов в `docs/templates/**`, проверить, нужно ли синхронизировать `external-docs/templates`.
- В приватные репозитории не коммитить секреты, токены, ключи и параметры реальных инсталляций.
- Если slug рабочего репозитория не начинается с `github.com/codex-k8s`, не считать приватные submodule `external-docs/private/**` доступными.
- Submodule использовать только для platform-owned source repos и публичных руководящих пакетов; устанавливаемые пакеты не делать submodule рабочего репозитория по умолчанию.
- Для пакетов использовать фиксированный `tag` или `commit`; для платных пакетов использовать только поставку через прокси платформы.
- Внешние пакеты поставщиков и клиентские пакеты не добавлять в `external-docs/**` без отдельного решения: по умолчанию они должны подключаться через пакетную платформу как источники установки, а не как submodule исходников.

## Источники

- Магазин пакетов: `external-docs/private/package-store`.
- Сайт и пользовательская документация: `external-docs/private/platform-site`.
- Общие гайды: `external-docs/guidelines/common`.
- Go-гайды: `external-docs/guidelines/go`.
- Vue-гайды: `external-docs/guidelines/vue`.
- Шаблоны документации: `external-docs/templates`.
