# Архив delivery-документов до рефакторинга

## TL;DR
- Здесь сохранены sprint/epic документы старой модели delivery.
- Эти материалы нужны только для исторической трассировки и анализа прошлых решений.
- Для новых волн 7+ используется wave-модель из `refactoring/task.md` и `refactoring/README.md`.

## Содержимое
- `docs/deprecated/pre-refactor/delivery/sprints/` — старые sprint-планы и sprint index.
- `docs/deprecated/pre-refactor/delivery/epics/` — старые epic/day-epic пакеты.

## Ограничения
- Не запускать новые задачи по старым sprint/day epic документам.
- Не создавать новые sprint/day epic файлы в этом архиве.
- Если нужно сослаться на прошлое решение, ссылка должна явно показывать, что это архив и только справочный материал.
