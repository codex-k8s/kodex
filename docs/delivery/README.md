# Delivery документация

## Назначение

`docs/delivery/**` хранит правила поставки, карты Issue, волны, трассируемость и приёмку новой версии платформы.

## Разделы

| Раздел | Назначение |
|---|---|
| `audits/` | Проверочные отчёты между волнами: технические риски, результаты аудита и краткая приёмка. |
| `coordination/` | Координация ролей поставки, текущая модель работы и архив прежней раскладки доменных агентов. |
| `issue-map/` | Разнесённые карты связей `Issue/PR ↔ документы ↔ волны`, без единого растущего файла. |
| `waves/` | Планы и статусы документационных и кодовых волн. |

## Сквозные планы

| Документ | Назначение |
|---|---|
| [mvp-roadmap.md](mvp-roadmap.md) | Маршрут до MVP: сквозной backend-срез, первый deploy, `staff-gateway`, web-console по черновым макетам и укрепление перед MVP. |

## Правила

- Не создавать один общий большой `issue_map.md`.
- Доменная работа обновляет только карту своего домена.
- Волновая работа обновляет только карту своей волны.
- Корневые индексы содержат навигацию и сводки, а не полные таблицы.
