# Индекс доменных агентов

## Назначение

Этот файл является стабильным индексом параллельных доменных агентов.

Здесь фиксируются только:

- номер агента;
- основной домен;
- основные сервисы;
- ссылка на личный файл координации агента;
- стабильная зона ответственности.

Текущий прогресс, временные переключения, блокировки, открытый бэклог и рекомендации следующего шага не размещаются в этом файле. Эти сведения каждый агент ведёт в своём личном файле, чтобы параллельные PR не конфликтовали в одном общем документе.

## Как читать координацию

- Сначала открыть этот индекс, чтобы понять владельца домена и нужный личный файл.
- Затем открыть личный файл агента, чтобы увидеть текущий прогресс, блокировки и следующий рациональный шаг.
- Если один агент обнаруживает блокировку от другого домена, он фиксирует её в своём личном файле и при необходимости просит владельца соседнего домена синхронизировать свой файл.
- Если меняется стабильная зона ответственности, номер агента, имя личного файла или добавляется новый агент, обновляется этот индекс.

## Правила обновления

- Не добавлять сюда статусы вида `готово`, `на ревью`, `в PR`, списки закрытых PR, временные переключения и детальные блокировки.
- Не дублировать бэклог личных файлов агентов.
- Не редактировать строки чужих агентов ради обновления их прогресса; такие изменения должны идти через личный файл соответствующего агента.
- Междоменные решения фиксировать в доменной документации и личных файлах затронутых агентов, а этот индекс обновлять только если изменилась стабильная раскладка ответственности.
