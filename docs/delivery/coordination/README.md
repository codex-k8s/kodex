# Координация доменных агентов

## Назначение

Раздел фиксирует текущую раскладку параллельной разработки по агентам, доменам и сервисам. Это документация поставки для синхронизации PR-срезов, а не замена доменной каноники.

## Документы

| Документ | Назначение |
|---|---|
| `agents.md` | Общая карта агентов, зон ответственности и известных блокировок между доменами. |
| `agent-1-project-catalog.md` | Краткий бэклог агента #1 по доменам проектов, репозиториев, runtime и fleet. |
| `agent-2-provider-hub.md` | Краткий бэклог агента #2 по домену provider-native интеграций. |
| `agent-3-package-hub.md` | Краткий бэклог агента #3 по домену пакетной платформы. |

## Правила обновления

- Каждый агент обновляет свой файл при изменении плана, появлении блокировки или снятии блокировки.
- Общую карту `agents.md` обновлять только кратко: кто чем владеет, где есть междоменные зависимости, какие решения требуют синхронизации.
- Если сведения по чужому агенту не подтверждены, помечать их как требующие подтверждения, а не выдавать за окончательный статус.
