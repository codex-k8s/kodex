# Координация работ и ролей

## Назначение

Раздел фиксирует рабочую модель координации поставки: кто принимает решения, кто проектирует границы, кто проверяет PR, кто отвечает за приёмку и кто ведёт эксплуатационные проверки. Это документация поставки для синхронизации рабочих срезов, а не замена доменной каноники.

Текущая активная модель для нового окружения — роли, а не постоянные пары из шести разработчиков и шести ревьюверов. Описанные правила являются драфтом перехода и должны уточняться новыми агентами по мере обкатки приложения. Старые файлы `agent-*.md` оставлены как архив фазы параллельной доменной разработки и справочный материал по уже выполненным срезам.

## Правила обновления

- Новые назначения фиксировать через GitHub Issues и ссылаться на роль из `roles.md`, а не на номер прежнего агента.
- Крупные междоменные решения сначала проверяет architect, после чего manager создаёт или уточняет задачи.
- Reviewer, QA и SRE не переписывают код в рамках проверки без отдельного назначения; они фиксируют замечания, статусы и безопасную диагностику в PR, Issue или отчёте.
- Старые `agent-*.md` не расширять для новых ролей. Их можно править только для архивной корректировки уже выполненной фазы.
- Если сведения по роли или чужому срезу не подтверждены, помечать их как требующие подтверждения, а не выдавать за окончательный статус.
