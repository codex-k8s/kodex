# Координация доменных агентов

## Назначение

Раздел фиксирует текущую раскладку параллельной разработки по агентам, доменам и сервисам. Это документация поставки для синхронизации PR-срезов, а не замена доменной каноники.

## Документы

| Документ | Назначение |
|---|---|
| `agents.md` | Стабильный индекс агентов, личных файлов и зон ответственности без текущих статусов и блокировок. |
| `agent-1-project-catalog.md` | Текущий прогресс, бэклог и блокировки агента #1 по доменам проектов, репозиториев, runtime, fleet и platform MCP. |
| `agent-2-provider-hub.md` | Текущий прогресс, бэклог и блокировки агента #2 по домену provider-native интеграций. |
| `agent-3-package-hub.md` | Текущий прогресс, бэклог и блокировки агента #3 по домену пакетной платформы и временным срезам `agent-orchestration`. |
| `agent-6-risk-governance.md` | Текущая зона, зависимости и стартовый срез агента #6 по домену рисков и релизов. |

## Правила обновления

- Каждый агент обновляет свой файл при изменении плана, появлении блокировки или снятии блокировки.
- Общий индекс `agents.md` обновлять только при изменении состава агентов, личных файлов или стабильной зоны ответственности.
- Текущий прогресс, временные переключения, междоменные блокировки и рекомендации следующего шага размещать только в личных файлах агентов.
- Если сведения по чужому агенту не подтверждены, помечать их как требующие подтверждения, а не выдавать за окончательный статус.
