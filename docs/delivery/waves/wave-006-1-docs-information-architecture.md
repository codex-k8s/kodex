# Wave 6.1 — информационная архитектура документации

## Цель

Собрать структуру `docs/**`, которая выдержит рост документации, параллельную работу агентов и будущий перенос частей документации в отдельные репозитории.

## Объём

- `docs/platform/**` — сквозная документация платформы.
- `docs/domains/**` — доменные пакеты.
- `docs/catalogs/**` — каталоги плагинов, руководящих пакетов и ролей.
- `docs/delivery/issue-map/**` — разнесённая карта Issue.
- `docs/research/**` — исследовательские материалы без статуса каноники.

## Критерии готовности

- Корневой индекс объясняет новую структуру.
- У каждого первичного домена есть `README.md`.
- Карта Issue разбита по доменам и волнам.
- `issue_map.md` используется как шаблон для отдельных файлов, а не как один общий реестр.
- `refactoring/24-pre-wave7-documentation-rebuild-plan.md` обновлён под выбранную структуру.
