# Доменные пакеты документации

## Назначение

`docs/domains/**` хранит самодостаточные пакеты по доменам платформы. Такой пакет должен быть достаточно независимым, чтобы позже его можно было перенести в отдельный репозиторий или отдельный пакет документации без переписывания всей навигации.

## Правила доменного пакета

- В корне домена всегда есть `README.md`.
- Документы домена не дублируют сквозную платформенную канонику из `docs/platform/**`.
- Если домену нужны продуктовые, архитектурные, delivery или ops документы, они создаются внутри домена.
- Карта Issue домена живёт в `docs/delivery/issue-map/domains/<domain>.md`.
- Волновая карта живёт в `docs/delivery/issue-map/waves/<wave>.md`.

## Первый набор доменов

| Домен | Назначение |
|---|---|
| `access-and-accounts/` | Доступ, организации, группы, пользователи и внешние аккаунты. |
| `projects-and-repositories/` | Проекты, репозитории, проектная документация и релизные политики. |
| `provider-native-work-items/` | `Issue`, `PR/MR`, комментарии, связи, ветки и теги провайдера. |
| `package-platform/` | Плагины, пакеты руководящей документации, установка, версии и каталоги. |
| `agent-orchestration/` | `agent-manager`, flow, stage, role, шаблоны промптов и automation rules. |
| `runtime-and-fleet/` | Слоты, `run`, `job`, серверы, кластеры и runtime manager. |
| `interaction-hub/` | MCP, согласования, уведомления, внешняя обратная связь и каналы взаимодействия. |
| `console-and-operations-ux/` | Операторская консоль и рабочие пространства. |
| `risk-and-release-governance/` | Риски, gates, release lines, branch rules и релизная автоматизация. |
| `billing-and-cost-accounting/` | Затраты, распределение расходов, счета и коммерческий контур. |
| `knowledge-lifecycle/` | Руководящая и проектная документация, самоулучшение и жизненный цикл знаний. |
