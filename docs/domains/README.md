# Доменные пакеты документации

## Назначение

`docs/domains/**` хранит самодостаточные пакеты по доменам платформы. Такой пакет должен быть достаточно независимым, чтобы позже его можно было перенести в отдельный репозиторий или отдельный пакет документации без переписывания всей навигации.

## Правила доменного пакета

- В корне домена всегда есть `README.md`.
- Документы домена не дублируют сквозную платформенную канонику из `docs/platform/**`.
- Если домену нужны продуктовые, архитектурные, delivery или ops документы, они создаются внутри домена.
- Карта Issue домена живёт в `docs/delivery/issue-map/domains/<domain>.md`.
- Волновая карта живёт в `docs/delivery/issue-map/waves/<wave>.md`.

## Первый набор доменов

| Домен | Назначение |
|---|---|
| `access-and-accounts/` | Доступ, организации, группы, пользователи, allowlist, внешние аккаунты как субъекты политики и аудит решений доступа. |
| `projects-and-repositories/` | Проекты, репозитории, проектная документация, правила веток, релизные политики и релизные линии. |
| `provider-native-work-items/` | `Issue`, `PR/MR`, комментарии, mentions, relationships, поля провайдера и platform watermarks. |
| `package-platform/` | Плагины, пакеты руководящей документации, клиент магазина пакетов, установка, версии и каталоги. |
| `agent-orchestration/` | `agent-manager`, flow, этапы, роли, шаблоны промптов и правила автоматизации. |
| `runtime-and-fleet/` | Слоты, `run`, `job`, серверы, Kubernetes-кластеры, размещение и platform jobs. |
| `interaction-hub/` | UI, голос, уведомления, внешняя обратная связь и расширяемые каналы взаимодействия. |
| `console-and-operations-ux/` | Командный центр, рабочие представления, операционные статусы и диагностические экраны. |
| `risk-and-release-governance/` | Риски, Human gate, релизные линии, правила веток и релизная автоматизация. |
| `billing-and-cost-accounting/` | Затраты, распределение расходов, счета и коммерческий контур. |
| `knowledge-lifecycle/` | Руководящая и проектная документация, проектные правила, самоулучшение и жизненный цикл знаний. |
