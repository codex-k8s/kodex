# Жизненный цикл знаний

## Назначение

Домен описывает шаблоны документов, проектные правила для агентов, пакеты руководящей документации, источники и структуру проектной документации в репозиториях проекта, загрузку проектной документации в рабочий контур агента вместе с кодом и руководящими пакетами, а также будущий переход к хранилищу знаний через векторные БД и MCP.

## Ключевые решения

- Руководящая документация живёт как пакет и описывает общие правила работы агентов.
- Проектная документация живёт в репозиториях проекта и описывается через `services.yaml`.
- Flow, профили ролей и шаблоны промптов живут в БД, а репозиторий платформы хранит только установочные фикстуры.
