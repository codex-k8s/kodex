# Пакетная платформа

## Назначение

Домен описывает плагины, пакеты руководящей документации, клиентские подключения к магазинам пакетов, каталоги, версии, установку, верификацию, права доступа и коммерческий статус пакетов.

## Связанные каталоги

- `docs/catalogs/package-store/`
- `docs/catalogs/plugins/`
- `docs/catalogs/guidance-packages/`

## Ключевые решения

- Каталог установленных пакетов хранится локально в платформе.
- Каталог доступных пакетов получает данные через клиентские подключения к магазинам пакетов.
- Магазин пакетов является устанавливаемым пакетом из репозитория-источника.
- Пакетная платформа не владеет магазином как бизнес-системой: она выступает клиентом магазина, хранит подключение, синхронизирует доступные пакеты и управляет локальными установками.
- Авторский магазин `kodex` живёт в приватном репозитории и не вшивается в ядро платформы.
- Сайт и документация платформы поставляются отдельным пакетом из отдельного репозитория.
