# Рабочие сущности провайдера

## Назначение

Домен описывает работу с нативными рабочими сущностями провайдера: `Issue`, `PR/MR`, комментариями, mentions, relationships, ветками, тегами, type, labels, milestones, полями проекта, метаданными провайдера, platform watermarks и открытыми инструкциями в body/comment слоях.

## Ключевые решения

- GitHub/GitLab остаётся источником рабочих артефактов.
- Инициатива — это `Issue` типа `initiative`, а не отдельная сущность платформы.
- Follow-up работа — это такие же `Issue` с другими типами.
- Платформа управляет оркестрацией, политиками, аудитом, приёмкой и проекциями вокруг нативных сущностей провайдера.
