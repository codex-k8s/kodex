# Магазин пакетов

## Назначение

Раздел описывает будущий магазин пакетов как устанавливаемый пакет, а не как жёстко встроенную часть ядра платформы.

У каждой установки платформы должен быть локальный каталог установленных пакетов и локальный каталог доступных пакетов. Каталог доступных пакетов получает данные через клиентские подключения к одному или нескольким магазинам пакетов.

## Каноническое решение

- Авторский магазин пакетов `kodex` поставляется из приватного репозитория как пакет.
- Мы устанавливаем этот пакет в собственную платформу и через него публикуем доступные пакеты.
- Другая установка платформы может использовать наш магазин, свой магазин или только пользовательские источники.
- Отдельный пакет из отдельного репозитория поставляет публичный сайт платформы и пользовательскую документацию платформы: API, разработка пакетов, добавление пакетов в магазин и каталог.

## Что не делать

- Не вшивать единственный магазин пакетов в ядро платформы.
- Не смешивать публичный сайт платформы с руководящими пакетами для агентов.
- Не считать каталог доступных пакетов источником состояния установки: состояние установки хранится локально в платформе.
