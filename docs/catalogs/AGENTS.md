# Инструкции для агентов в `docs/catalogs/**`

## Назначение

`docs/catalogs/**` описывает каталоги платформы: пакеты, плагины, пакеты руководящей документации, роли агентов, процессы `flow` и шаблоны промптов.

## Перед изменениями

- Читать `docs/AGENTS.md`.
- Читать `docs/catalogs/README.md`.
- Читать `refactoring/17-plugin-system-and-package-market.md`, если изменение связано с пакетами, магазином пакетов или плагинами.
- Читать `refactoring/18-knowledge-package-management.md`, если изменение связано с руководящей или проектной документацией.
- Читать `docs/templates/index.md` и выбирать шаблон под тип создаваемого документа.

## Разделы

- `docs/catalogs/package-store/**` — магазин пакетов как устанавливаемый пакет и источник доступных пакетов.
- `docs/catalogs/plugins/**` — правила публикации, проверки, установки и эксплуатации плагинов.
- `docs/catalogs/guidance-packages/**` — пакеты руководящей документации для агентов.
- `docs/catalogs/prompt-roles/**` — встроенные и пользовательские роли, процессы `flow` и шаблоны промптов.

## Правила содержания

- Магазин пакетов является отдельным пакетом платформы из отдельного приватного репозитория; ядро платформы и `package-platform` работают с ним как с внешним источником данных и установок.
- Каталог установленных пакетов есть у каждой инсталляции платформы, а каталог доступных пакетов может получать данные из магазина пакетов.
- Пользовательская документация платформы, например API и правила разработки пакетов, поставляется отдельным пакетом сайта и документации.
- Текст шаблонов промптов, процессов `flow` и руководящих пакетов хранится в репозиториях-источниках, а не как свободный текст в БД.
- Не смешивать проектную продуктовую документацию с руководящими пакетами для агентов.
