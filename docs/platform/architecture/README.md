# Архитектура платформы

## Назначение

Раздел хранит сквозную архитектуру всей платформы. Доменная архитектура размещается в `docs/domains/<domain>/architecture/`.

## Планируемые документы и шаблоны

| Документ | Шаблон | Смысл |
|---|---|---|
| C4-контекст | `c4_context.md` | Показывает платформу, внешние системы и основных участников. |
| C4-контейнеры | `c4_container.md` | Показывает целевые сервисы и зоны ответственности. |
| Проектное решение | `design_doc.md` | Описывает сквозную архитектурную модель. |
| Модель данных | `data_model.md` | Фиксирует основные сущности, владение данными и связи. |
| API-контракты | `api_contract.md` | Описывает подход к HTTP, gRPC, MCP и доменным событиям. |
| Архитектурные решения | `adr.md` | Фиксирует принятые решения и их последствия. |
| Альтернативы | `alternatives.md` | Показывает рассмотренные варианты и причины выбора. |
| Политика миграций | `migrations_policy.md` | Фиксирует правила изменения схем и данных. |
