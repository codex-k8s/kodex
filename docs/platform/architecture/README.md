# Архитектура платформы

## Назначение

Раздел хранит сквозную архитектуру всей платформы. Доменная архитектура размещается в `docs/domains/<domain>/architecture/`.

Архитектурный пакет задаёт целевую рамку для кодовой реализации: сервисные границы, владение данными, контракты, интеграцию с провайдерами и платформенный MCP-контур.

## Активная каноника

| Документ | Назначение |
|---|---|
| `c4_context.md` | Контекст платформы: участники, внешние системы и границы ответственности. |
| `c4_container.md` | Контейнерная карта: edge, доменные сервисы, исполнители, хранилища и внешние зависимости. |
| `domain_map.md` | Архитектурная карта доменов и сервисов-владельцев. |
| `service_boundaries.md` | Правила владения сервисов, edge-компонентов и исполнителей. |
| `data_model.md` | Сквозная модель данных, владение состоянием и межсервисная синхронизация. |
| `provider_integration_model.md` | Модель интеграции с GitHub/GitLab и другими провайдерами. |
| `mcp_and_interaction_model.md` | Платформенный MCP, взаимодействия с человеком, уведомления и внешние каналы. |

## Правила развития

- Сквозные решения, которые затрагивают несколько доменов, фиксируются здесь.
- Детали конкретного домена выносятся в `docs/domains/<domain>/architecture/**`.
- Документы этого раздела не должны ссылаться в теле на временные PR, review threads или номера рабочих задач.
- Если меняется граница сервиса-владельца, нужно синхронно обновить `domain_map.md`, `service_boundaries.md` и `data_model.md`.
- Если меняется provider-контур, нужно синхронно обновить `provider_integration_model.md` и контракты домена `provider-native-work-items`.
