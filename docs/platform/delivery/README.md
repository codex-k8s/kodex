# Delivery платформы

## Назначение

Раздел хранит общие правила поставки платформы: как идут волны, как принимаются PR, как проверяется трассируемость и какие артефакты обязательны.

Детальные карты Issue размещаются в `docs/delivery/issue-map/`.

## Live-safe изменения

Платформа уже развёрнута в live-кластере. Поставка быстрых внутренних изменений остаётся допустимой,
но миграции, секреты, конфиги, транспортные контракты и deploy path должны иметь последовательную
историю обновлений:

- уже применённые миграции не редактируются; изменения схемы идут новой forward-only additive migration;
- destructive/drop/rename выполняются только через путь `expand -> migrate -> contract`;
- breaking change в `proto`, OpenAPI или AsyncAPI допускается только при согласованном обновлении всех
  потребителей без live rolling gap; иначе применяется staged путь `add -> migrate consumers -> remove`;
- новые обязательные env/secrets/config получают staged rollout path или безопасную диагностику;
- deploy tooling сохраняет возможность последовательного обновления существующего live-кластера.

## Планируемые документы и шаблоны

| Документ | Шаблон | Смысл |
|---|---|---|
| План поставки | `delivery_plan.md` | Описывает порядок волн и условия перехода между ними. |
| Definition of Done | `definition_of_done.md` | Фиксирует критерии готовности документа, домена, PR и волны. |
| Дорожная карта | `roadmap.md` | Показывает последовательность больших направлений. |
| Карта Issue | `issue_map.md` | Используется как шаблон для доменных и волновых карт, а не как один общий файл. |
| Задача на пакет документов | `docset_issue.md` | Описывает, как заводить работу на документационный пакет. |
| PR пакета документов | `docset_pr.md` | Описывает, как принимать документационный пакет. |
