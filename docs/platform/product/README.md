# Продуктовая документация платформы

## Назначение

Раздел хранит сквозные продуктовые документы `kodex`: краткое описание продукта, ограничения, продуктовую модель, терминологию и требования на уровне всей платформы.

В рамках wave 6.2 этот раздел становится первым активным переносом продуктовой каноники из `refactoring/**` в новую структуру `docs/**`. Если между кратким сквозным документом здесь и более детальным программным документом из `refactoring/**` возникает расхождение, до явного переноса соответствующей темы приоритет остаётся у `refactoring/task.md` и связанных документов программы.

## Активные документы

| Документ | Назначение |
|---|---|
| `brief.md` | Краткая продуктовая рамка платформы: проблема, пользователи, решение и границы. |
| `constraints.md` | Неподвижные продуктовые, технические, операционные и документационные ограничения. |
| `product_model.md` | Сквозная модель сущностей и рабочих контуров платформы. |
| `glossary.md` | Канонические термины для активной документации и будущей реализации. |
| `requirements.md` | Сквозные функциональные и нефункциональные требования платформы. |

## Планируемые документы и шаблоны

| Документ | Шаблон | Смысл |
|---|---|---|
| Устав проекта | `project_charter.md` | Фиксирует цель платформы, владельца, рамки и способ принятия решений. |
| Проблема | `problem.md` | Описывает боль, которую решает платформа. |
| Краткое описание | `brief.md` | Создано в `brief.md`. |
| Ограничения | `constraints.md` | Создано в `constraints.md`. |
| Персоны | `personas.md` | Описывает пользователей, операторов и организации. |
| Рамки MVP | `scope_mvp.md` | Отделяет первую версию от enterprise-заделов. |
| Требования | `prd.md` | Создано в `requirements.md`. |
| Нефункциональные требования | `nfr.md` | Описывает безопасность, аудит, устойчивость и производительность. |
| Метрики успеха | `success_metrics.md` | Показывает, как понять, что цель достигнута. |

## Правила развития

- Детали конкретного домена не добавляются в этот раздел напрямую: их место в `docs/domains/<domain>/**`.
- Термины из `glossary.md` используются во всех новых продуктовых, архитектурных и delivery-документах.
- Если следующий документ уточняет требования, он должен ссылаться на `requirements.md`, а не дублировать список требований целиком.
