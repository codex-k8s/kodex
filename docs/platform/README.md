# Документация платформы

## Назначение

`docs/platform/**` хранит сквозную документацию всей платформы `kodex`: что строим, какие системные ограничения принимаем, как устроена целевая архитектура, как поставляем изменения и как эксплуатируем платформу.

Этот раздел не должен превращаться в склад документов по отдельным доменам. Если документ относится к конкретному домену, он должен жить в `docs/domains/<domain>/`.

## Разделы

| Раздел | Назначение |
|---|---|
| `product/` | Сквозная продуктовая модель платформы. |
| `architecture/` | Общая архитектура и контейнерная карта платформы. |
| `delivery/` | Общие правила поставки, приёмки и управления волнами. |
| `ops/` | Сквозные эксплуатационные правила платформы. |

## Активная каноника

| Документ | Назначение |
|---|---|
| `product/brief.md` | Краткая продуктовая рамка платформы. |
| `product/constraints.md` | Неподвижные ограничения платформы. |
| `product/product_model.md` | Сквозная продуктовая модель платформы. |
| `product/glossary.md` | Продуктовый глоссарий платформы. |
| `product/requirements.md` | Сквозные продуктовые требования. |

## Что сюда попадёт позже

- устав проекта;
- C4-контекст и C4-контейнеры;
- общая модель данных;
- общие правила поставки;
- общий runbook платформы.

## Что сюда не класть

- документы конкретного домена;
- карты Issue по домену или волне;
- документацию конкретного сервиса;
- руководящие пакеты для агентов;
- старые материалы из `deprecated/**`.
