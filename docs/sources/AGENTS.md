# Инструкции для агентов в `docs/sources/**`

## Назначение

`docs/sources/**` содержит необязательные submodule-ссылки на внешние репозитории документации. Это не место для редактирования активной каноники `docs/**` напрямую.

Активная документация должна жить в `docs/**` без отдельного параллельного корня. Если нужен внешний репозиторий-источник, он подключается под `docs/sources/**`, а не в новую верхнеуровневую папку.

## Правила работы

- Обычная работа в `kodex` не должна требовать инициализации submodule.
- Не запускать массовый `git submodule update --init --recursive` без причины.
- Если меняется содержимое внешнего репозитория, сначала commit/push делается внутри этого внешнего репозитория, затем в `kodex` обновляется только gitlink.
- Изменения в содержимом внешнего репозитория вносятся через отдельную ветку и PR этого репозитория; обновление gitlink в `kodex` делается уже после merge.
- Если меняется активная локальная копия гайдов в `docs/design-guidelines/**`, проверить, нужно ли синхронизировать соответствующий публичный репозиторий в `docs/sources/guidelines/**`.
- Если меняется активная локальная копия шаблонов в `docs/templates/**`, проверить, нужно ли синхронизировать `docs/sources/templates`.

## Источники

- Общие гайды: `docs/sources/guidelines/common`.
- Go-гайды: `docs/sources/guidelines/go`.
- Vue-гайды: `docs/sources/guidelines/vue`.
- Шаблоны документации: `docs/sources/templates`.
