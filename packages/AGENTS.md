# Инструкции для агентов в `packages/**`

## Назначение

`packages/**` содержит необязательные submodule-ссылки на исходники платформенных пакетов. Это не основной путь установки пакетов в платформу, а только рабочий контур сопровождения platform-owned source repos.

## Правила работы

- Обычная работа в `kodex` не должна требовать инициализации `submodule` из `packages/**`.
- Приватные `submodule` инициализировать только при явной необходимости и наличии доступа.
- Не запускать массовый `git submodule update --init --recursive` без причины.
- Если меняется содержимое пакетного репозитория, сначала изменения вносятся в его собственной ветке и PR, и только после merge в `kodex` обновляется gitlink.
- В приватные пакетные репозитории не коммитить секреты, токены, ключи и параметры реальных инсталляций.
- Если slug рабочего репозитория не начинается с `github.com/codex-k8s`, не считать `packages/**` доступными по умолчанию.
- `packages/**` использовать только для platform-owned source repos.
- Устанавливаемые пакеты, внешние пакеты поставщиков и клиентские пакеты не добавлять в `packages/**` без отдельного решения.

## Источники

- Магазин пакетов: `packages/package-store`.
- Сайт и пользовательская документация платформы: `packages/platform-site`.
